Genetic Programming

blackdogtop
2020-05-30

What is genetic programming? Describe its history briefly.

Segaran (2007) stated genetic programming is a machine-learning technique which is inspired by biological evolution and it starts with a lot of randomly generated or hand-designed programs (population) to compete in some user- defined task i.e. we can think of it as a game, the purpose is to compare which program has better performance. Meanwhile, Koza & Koza (1992) pointed out that genetic programming combines Darwinian natural selection and genetic manipulation, that is, offspring programs are generated by crossover between current programs and the individuals who adapt to the environment are preserved and prosperous.

I think genetic programming is different from other machine learning tech- niques that I know. First, the most machine learning techniques that we learn is to tell the program how to solve it according to the question i.e. we provide algorithms for the program to find the best parameters. Therefore, the program does not have the ability to choose independently, and always solves the problem according to the algorithm we set. At the same time, we need to recreate different algorithms for different problems. However, genetic programming tells a lot of different programs about the problem without assign specific and well-performing algorithms to each program i.e. the algorithm of each program is generated by the program itself. Second, genetic algorithm not only optimizes the algorithm through parameter adjustment, but also generates new programs through crossover and mutation, and retains the programs that perform well through natural selection.

Cramer (1985) published first statement of modern “tree-based” Genetic Programming.

Koza (1989) published in the International Joint Conference on Artificial Intelligence IJCAI-89 and Koza (1990) patented his invention of a GA for program evolution.

Koza & Koza (1992) published that really established genetic programming. Koza (2010) proposed that genetic programming was human competitive.

How is it related to the genetic algorithms that we discussed before?

According to Affenzeller et al. (2009), genetic programming is an extension of genetic algorithm, and both of genetic programming(GP) and genetic algorithm(GA) are based on Darwinian principles of survival of the fittest (selection), recombination (crossover), and mutation. We can infer that for both GP and GA, select good individuals from the current individuals as parents. To be specific, each generation forms a new generation through mutation and crossover, and individuals with good fitness function performance are selected as parents as much as possible.

On the other hand, although GA and GP are very similar, there are different. The difference between GA and GP is first reflected in the individual. The individual of GP is a program, and individual of GA is a fixed-length gene to represent a series of values. Like the pizza example that we mentioned in lecture, each recipe of pizza is an individual and represented by a string of 10 binary digits. Furthermore, because of the fixed-length gene, the functions embodied by genes are not very complicated. i.e. the pizza with 10 ingredients can be combined to seem to have many ($2^10$ = 1024) possibilities, but it is far from the complexity of the process of natural evolution to be simulated. Meanwhile, we abstract the evolutionary process of biology into a mathematical model. When crossover or mutation GA genes, operators probably lead to invalid states. e.g. ‘a ∗ b’ use some kind of crossover or mutation to generate ‘a + /c’ - it is obviously an invalid state. Therefore, when performing genetic algorithms, we need to always pay attention to the invalid genes caused by operators.

The length (depth) of each individual of GP is generally different and the encoding of GP allows variable length unlike GA with fixed length. Meanwhile, Segaran (2007) proposed that there are many ways for representing genetic programming, tree representation is most commonly used, the node of a tree probably be a very complex function such as Gaussians, and we can construct the tree recursively. Therefore, compared with GA, it can express the nature of complex things more flexibly.

What kind of genotype representations are used in GP?

According to Weise (2009), Genetic Programming is divided into three types according to the genotype, Linear Genetic Programming, Tree-Based Genetic Programming and Graph-Based Genetic Programming.

First, for Linear Genetic Programming, it has Generalized Linear Genetic Programming and Narrow Linear Genetic Programming.

For Generalized Linear Genetic Programming, the genotype is a linear string and such Multi-Expression Programming, Cartesian Genetic Programming etc are included. On the other hand, the individual of Narrow Linear Genetic Programming can be an assembly language or high-level programming language.

Perkis (1994) gave an example of Linear Genetic Programming is stack-based GP. Each program instruction gets its parameters from the stack, after performs its calculations, pushes the results back onto the stack.

Next, for Tree-Based Genetic Programming, Segaran (2007) emphasised tree representation is commonly used, and it has two types of nodes, which are child nodes and an endpoint. They respectively represent an operation applied to its child nodes and a parameter with constant value. For example, the rectangular node is a subtraction operation on its two branches(Y and 2) based on the premise that the left branch is not true. Then the result will be given to the node above. It can easily modify the sub-tree by using Tree-Based Genetic Programming. Meanwhile, Tree-based Genetic Programming also has mutation and crossover and retain the syntactic validity of the expression.

Last, for Graph-Based Graph-based Programming, Galvan-Lopez (2008) gave an example of program graph, and we can think of it as a circuit, the different nodes represent different typical operators (i.e. AND, OR, etc.). We can notice the structure of the program graph is similar to the structure of tree that we reference below. Because technically the tree is a special kind of graph. Therefore, we can infer the genotype representation is similar to the last tree one.
